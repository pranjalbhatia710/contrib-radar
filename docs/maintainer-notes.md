# Maintainer notes

`contrib-radar` is designed to reduce maintainer noise, not generate drive-by pull requests.

## Recommended contributor workflow

1. Pull the top-ranked issues for a repository.
2. Read the issue and linked discussion manually.
3. Check whether a related pull request already exists.
4. Make the smallest useful change that closes one issue.
5. Include validation output in the PR body.

## What the score does not decide

The score is a triage hint. It does not replace judgment about:

- project priorities
- maintainer bandwidth
- whether the issue is still valid
- whether the contributor has enough context

If a high-scoring issue already has a clear owner or an active PR, skip it.
